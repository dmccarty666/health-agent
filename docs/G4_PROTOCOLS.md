## Story G4.1 — Goal-Based Health Protocols

**Points:** 4 | **Owner:** dev | **Status:** Proposed | **Prerequisite:** G2.1, G2.2, G2.4

**What:** Create protocol cards — goal-driven recommendations tied to personal data. Each protocol has a goal, data basis, recommendation, what to track, re-test date, and confidence level. Generated by Hermes with human review.

**Tasks:**
- [ ] Build protocol data model (goal, data_basis, recommendation, tracking_metric, retest_date, confidence, status)
- [ ] Create protocol CRUD endpoints
- [ ] Build Hermes-powered protocol suggestion — "Given my data, what protocols should I follow?"
- [ ] Build protocol card UI on Hermes tab
- [ ] Add protocol status tracking (active, completed, paused)
- [ ] Add confidence scoring (strong signal, possible signal, insufficient data)

**Acceptance Criteria:**

| # | Criterion | Test |
|---|---|---|
| G4.1-AC1 | Hermes generates protocol from user's health data | integration |
| G4.1-AC2 | Protocol card shows goal, recommendation, tracking, retest date, confidence | smoke |
| G4.1-AC3 | Protocol status can be changed (active → completed → paused) | integration |
| G4.1-AC4 | Confidence score reflects data quality (more data = higher confidence) | unit |
| G4.1-AC5 | Multiple active protocols display without layout break | smoke |

**Definition of Done:** Hermes can generate personalized protocols from data. Protocol cards are actionable with tracking and retest dates. Confidence scoring is transparent.

## Story G4.3 — Training & Nutrition Periodization

**Points:** 3 | **Owner:** dev | **Status:** Proposed | **Prerequisite:** G1.5, G1.6, G2.1, G2.2

**What:** Analyze training load trends and diet patterns to suggest periodization. Detect overreaching signals. Recommend macro adjustments for current training phase (bulk, cut, maintenance). Deload suggestions based on recovery trends.

**Tasks:**
- [ ] Build training load analysis — detect increasing/decreasing trends
- [ ] Build overreaching detection — sustained high load + declining recovery
- [ ] Generate macro recommendations based on training phase and body comp trends
- [ ] Build deload suggestion trigger (declining readiness + sustained high load)
- [ ] Build periodization view on Training tab

**Acceptance Criteria:**

| # | Criterion | Test |
|---|---|---|
| G4.3-AC1 | Training load trend detected and visualized | integration |
| G4.3-AC2 | Overreaching flagged when load ↑ and recovery ↓ | unit |
| G4.3-AC3 | Macro recommendations align with training phase + body comp trend | integration |
| G4.3-AC4 | Deload suggestion triggered by multi-signal decline | unit |
| G4.3-AC5 | Periodization view is readable and actionable | smoke |

**Definition of Done:** Training load and recovery analyzed together. Periodization recommendations generated. Overreaching detected before burnout.

- [ ] Build PDF report generator (labs table, supplement list, training summary, diet summary, questions)

- [ ] Schedule monthly report cron (1st of month)
- [ ] Build report viewer UI on Hermes tab

| # | Criterion | Test |
